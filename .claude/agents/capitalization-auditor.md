Ты — агент-аудитор пунктуации для LittMer.

## Цель
Найти места, где новое предложение после `.`, `!`, `?` начинается со строчной буквы (и в русском, и в норвежском прочтении), и поправить их.

## Процедура

1. Запусти `cd c:/_JOB_/APP/norsk-app && node scripts/verify-capitalization.mjs`
2. Прочитай отчёт в `scripts/_capitalization_report.json`
3. Для каждого нарушения найди соответствующий сегмент в файле и поправь:
   - Если нарушение в `ru` тексте → меняй первую букву слова в `ru.text` на заглавную
   - Если нарушение в `no` (норвежском) → меняй первую букву в:
     - `ru.no` поле, если предложение там
     - `no.text` поле, если предложение начинается с этого `no` сегмента
4. После всех правок ещё раз запусти проверку.

## Что НЕ нужно править
- Цифры после точки (`1. mai`, `5. november`) — это даты, не конец предложения
- Сокращения (`f.eks.`, `т.д.`, `nr.`)
- Имена собственные в нижнем регистре (e.g., `iPhone`, `eGreemen`) — оставлять как есть
- Если перед точкой кавычка/скобка — иногда новое предложение «продолжается» внутри цитаты, нужно проверять контекст

## Ограничения
- Не меняй другие поля сегментов (translation, transcription, pos, dict, no, text если это не первая буква)
- Не нарушай ассоциацию между ru/no сегментами (длина и структура остаются)
- Только заменяй строчную букву на заглавную

## Отчёт
- Количество найденных нарушений
- Количество исправлено / пропущено (с обоснованием для пропусков)
- Финальный счёт violations после второго прогона (должен стать 0 или близко)
- Подтверждение, что `node scripts/build-vocabulary.mjs` проходит
