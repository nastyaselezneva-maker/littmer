# NorskApp — План проекта

Сайт для изучения норвежского языка через гибридные тексты: русский текст с вставками норвежских слов.
Уровни сложности: A2–B2. Разные темы.

---

## Фаза 1. Фундамент

### 1.1 Структура проекта
**Что делаем:** Организуем папки и файлы так, чтобы проект было легко развивать.
**Зачем:** Хорошая структура — основа масштабируемого проекта. Без неё код быстро превращается в кашу.

```
src/
  components/    — переиспользуемые UI-компоненты (кнопки, карточки, тултипы)
  pages/         — страницы приложения (главная, каталог, чтение, словарь)
  data/          — тексты и словари (пока просто JSON-файлы)
  styles/        — глобальные стили
  App.jsx        — корневой компонент
  main.jsx       — точка входа
```

**Что изучим:** как React-приложение организовано изнутри, зачем разделять код по папкам.

---

### 1.2 Роутинг (навигация между страницами)
**Что делаем:** Подключаем React Router — библиотеку для навигации.
**Зачем:** Чтобы у приложения были отдельные страницы с адресами в браузере.

Страницы:
- `/` — главная (приветствие, выбор уровня)
- `/texts` — каталог текстов
- `/texts/:id` — страница чтения конкретного текста
- `/dictionary` — личный словарь

**Что изучим:** как работает роутинг в SPA (Single Page Application), что такое динамические маршруты (`:id`).

---

### 1.3 Layout (общий каркас)
**Что делаем:** Создаём шапку с навигацией и обёртку для страниц.
**Зачем:** Шапка и навигация одинаковы на всех страницах — выносим их в общий компонент.

**Что изучим:** композиция компонентов, `children` в React, `<Outlet>` в React Router.

---

## Фаза 2. Данные и модель текстов

### 2.1 Формат данных текста
**Что делаем:** Придумываем JSON-структуру для хранения гибридных текстов.
**Зачем:** Нужен удобный формат, чтобы отличать русские части от норвежских.

Пример формата:
```json
{
  "id": "text-001",
  "title": "В магазине",
  "level": "A2",
  "topic": "shopping",
  "segments": [
    { "type": "ru", "text": "Я пошёл в " },
    { "type": "no", "text": "butikken", "translation": "магазин", "transcription": "бутиккен" },
    { "type": "ru", "text": " и купил " },
    { "type": "no", "text": "melk", "translation": "молоко", "transcription": "мэльк" }
  ]
}
```

**Что изучим:** как проектировать структуры данных, почему формат данных важнее кода.

---

### 2.2 Демо-данные
**Что делаем:** Создаём 3–5 текстов разных уровней и тем.
**Зачем:** Без данных нечего показывать — нужен контент для тестирования.

**Что изучим:** работа с JSON, импорт данных в React.

---

## Фаза 3. Страницы

### 3.1 Главная страница
**Что делаем:** Простая страница с описанием проекта и кнопками перехода к текстам.
**Зачем:** Точка входа для пользователя.

**Что изучим:** JSX, базовая стилизация, навигация через `<Link>`.

---

### 3.2 Каталог текстов
**Что делаем:** Список карточек с текстами. Фильтры по уровню и теме.
**Зачем:** Пользователь должен легко найти текст нужной сложности.

Что будет на странице:
- Фильтр по уровню: A2 / B1 / B2
- Фильтр по теме: магазин, семья, путешествия и т.д.
- Карточки текстов: название, уровень, тема, краткое описание

**Что изучим:** `useState` для фильтров, `.filter()` и `.map()` для списков, пропсы компонентов.

---

### 3.3 Страница чтения текста
**Что делаем:** Отображаем гибридный текст — русские слова обычным шрифтом, норвежские выделены.
**Зачем:** Это ключевая страница — здесь происходит обучение.

Функции:
- Норвежские слова визуально выделены (цвет, подчёркивание)
- Клик по норвежскому слову — показ перевода и транскрипции (тултип)
- Кнопка "Добавить в словарь" в тултипе

**Что изучим:** условный рендеринг, обработка событий, `useParams` для получения ID текста из URL.

---

### 3.4 Страница личного словаря
**Что делаем:** Таблица/список сохранённых слов с переводом и транскрипцией.
**Зачем:** Пользователь собирает свой набор слов для повторения.

Функции:
- Список слов: норвежское слово, перевод, транскрипция
- Удаление слова из словаря
- Данные хранятся в localStorage (сохраняются между сессиями)

**Что изучим:** `localStorage`, `useEffect`, управление состоянием списка.

---

## Фаза 4. Интерактив и полировка

### 4.1 Тултипы
**Что делаем:** При клике на норвежское слово — всплывающая подсказка с переводом.
**Зачем:** Не нужно покидать текст, чтобы узнать значение слова.

**Что изучим:** позиционирование элементов, управление видимостью, обработка кликов.

---

### 4.2 Стилизация
**Что делаем:** Приводим дизайн в порядок — отступы, цвета, типографика.
**Зачем:** Приятный интерфейс мотивирует учиться.

**Что изучим:** CSS, адаптивная вёрстка, основы UX.

---
